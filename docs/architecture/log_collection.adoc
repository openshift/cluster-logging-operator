== Log Collection (and Forwarding)

=== Overview



The ability to collect logs in an OpenShift cluster, be it container logs from application workloads,
journal logs from node services, or audit logs generated by the cluster API servers, requires the
collection agent to have direct, physical access to the source.  This means the collector MUST mount
the host file system to read files from the node; mounting the node host requires elevated permissions.

Each of the mentioned sources are file based which means the collector has no mechanism to impose backpressure
on the source when it is unable to forward logs to the intended destinations.
